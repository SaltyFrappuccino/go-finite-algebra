# Таблицы Кэли, поля и V*

Главная идея простая - всё считается напрямую по таблицам операций. Готовых библиотек для алгебры, конечных полей или линейной алгебры здесь нет. Внешних зависимостей у проекта тоже нет.

## Что именно считается

Все структуры строятся на размеченном множестве `{0, ..., n-1}` для порядков `2`, `3`, `4`.

- Полугруппа: одна ассоциативная бинарная операция.
- Моноид: полугруппа с двусторонней единицей.
- Группа: моноид, где у каждого элемента есть двусторонний обратный.
- Полукольцо: `+` - коммутативный моноид, `*` - полугруппа, выполняются обе дистрибутивности, ноль по `+` поглощает умножение.
- Полукольцо с единицей: то же самое, но `*` дополнительно имеет двустороннюю единицу. Считается отдельной колонкой, чтобы результат подходил под обе принятые трактовки термина.
- Кольцо: полукольцо, где `+` является абелевой группой.
- Кольцо с единицей: кольцо, где `*` имеет двустороннюю единицу.
- Тело: кольцо с единицей, где ненулевые элементы образуют группу по умножению.
- Поле: коммутативное тело.

Для второй части строится пространство `V = F^n` над полем `F2`, `F3` или `F4` при размерностях `1..4`. Флаги `-field` и `-dim` выбирают пример для печати в консоль, а проверки и CSV-статистика считаются по всем 12 сочетаниям. Проверяются базисы, координаты, кодирование и декодирование вектора, матрица перехода базисов, матрица пересчёта координат, сопряжённый базис в `V*`, ковекторы и изотропные векторы для формы `q(v)=sum xi^2`.

`F4` здесь - это `GF(4)`, поле из четырёх элементов. Это не `Z/4`: в `Z/4` есть делитель нуля `2*2=0`, поэтому `Z/4` не является полем.

## Как это проверяется

Ассоциативные таблицы строятся перебором с отсечениями по условию `(ab)c = a(bc)`. Пары операций проверяются прямой подстановкой аксиом на всех элементах.

Для полей дополнительно перебираются все перестановки элементов: так проверяется изоморфизм и считается число автоморфизмов. Поэтому видно, почему для порядка `4` получается `12` размеченных полей: `4! / |Aut(F4)| = 24 / 2 = 12`.

Линейная алгебра считается обычным методом Гаусса, но над конечным полем, а не над `float64`.

Для базисов используется такое соглашение: матрица перехода базисов `C(A<-B)` составлена из столбцов нового базиса `B`, записанных в старом базисе `A`. Поэтому `[v]_A = C(A<-B)[v]_B`, а координаты из `A` в `B` пересчитываются обратной матрицей: `[v]_B = C(A<-B)^-1[v]_A`.

## Запуск

```powershell
go test ./...
go run .\cmd\nct -out out -field 3 -dim 3
```

Параметры:

- `-field`: поле `2`, `3` или `4`.
- `-dim`: размерность от `1` до `4`.
- `-out`: папка для файлов результата.

## Что смотреть после запуска

Программа печатает все таблицы и проверки прямо в консоль и складывает данные в `out/`:

- `out/algebra_counts.csv` - численные итоги по всем порядкам.
- `out/vector_spaces.csv` - статистика по всем `F2/F3/F4` и `dim=1..4`.
- `out/verification.json` - 129 проверок-инвариантов в машинно-читаемом виде.
- `out/fields_order_2.json`, `out/fields_order_3.json`, `out/fields_order_4.json` - все найденные пары таблиц Кэли для полей.
